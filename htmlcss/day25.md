# Day 25 – User Login and User-Specific Todo Management

**Date:** Day 25

## Objective

To enhance the Todo application by planning and implementing a simple user login system and understanding how to display user-specific data.

## Topics Covered

- Introduction to User Authentication
- Login Page Design using React
- User Model in PostgreSQL
- Relationship between Users and Todos
- Foreign Key (user_id)
- User-Specific Data Management
- Frontend and Backend Integration
- React Components
- API Communication using Axios
- CRUD Operations for Individual Users

## Practical Work

- Designed a simple Login page for the Todo application.
- Planned the User table in PostgreSQL.
- Modified the Todo model to include a **user_id** field.
- Understood how each todo is linked to a specific user.
- Discussed creating separate Login and Todo pages in React.
- Planned APIs for user login and fetching user-specific todos.
- Improved the Todo application's user interface with modern styling.
- Explored how React communicates with FastAPI for authentication and todo management.

## Technologies Used

- React
- FastAPI
- PostgreSQL
- SQLAlchemy
- Axios
- CSS

## Learning Outcomes

- Understood the concept of user authentication.
- Learned how a **Foreign Key (user_id)** connects users with their todos.
- Learned how to display only the logged-in user's tasks.
- Understood the workflow of frontend-backend authentication.
- Improved understanding of React component design and API integration.

## Status

**Planned and implemented the basic structure for a login-enabled Todo application with user-specific task management**
